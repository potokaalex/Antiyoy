как работает cellFactory ? 
    1) создаёт клетки при загрузке прогресса = регистрация фабрики -> вызов загрузки прогресса -> создание карты
        плюсы: фабрика сама себя регистрирует (т.е. фабрика получает в зависимость saveLoader, что подразумевает самостоятельную загрузку прогресса - не аргумент!)
        минусы: неочевидность загрузки карты, возможные конфликты с другими объектами которые требуют карту для загрузки.
    2) создаёт клетки основываясь на прогрессе который передаётся в метод Create = внешне получаю нужный прогресс и запихиваю его в фабрику
        плюсы: фабрика знает только о нужном для себя прогрессе и не получает никакие другие сервисы.
        минусы: дополнительные затраты на засовывание данных в метод 
    3) создаёт клетки основываясь на прогрессе который передаётся через progressProvider
        плюсы: автономность, нет необходимости запихивать данные в фабрику.
        минусы: неочевидность, фабрика неочевидно зависит от целого progressProvider`а!
    итог: второй способ
